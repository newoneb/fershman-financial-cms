Contentful CMS Setup

Follow these steps to set up your Contentful content models:

1. Create a Contentful Account

If you don't have one already, sign up at contentful.com.

2. Create a New Space

Once logged in, create a new space for your Fershman Financial content.

3. Set Up Content Models

Create the following content models:

Article Model

  • Title (Short text)
  • Slug (Short text) - URL-friendly version of the title
  • Category (Short text with predefined values: Markets, Analysis, Policy & Regulation, Opinion)
  • Author (Reference to Author content type)
  • Publication Date (Date & time)
  • Excerpt (Long text) - Short summary for previews
  • Content (Rich text) - Main article content
  • Featured Image (Media) - Main image for the article
  • Featured (Boolean) - Whether to show on homepage
  • Premium (Boolean) - Whether it's premium content
  • Tags (Short text, list) - Related keywords
  • Meta Description (Long text) - For SEO

Author Model

  • Name (Short text)
  • Title (Short text) - Job title or position
  • Bio (Long text)
  • Image (Media) - Author photo

Market Data Model

  • Symbol (Short text) - e.g., "S&P 500"
  • Value (Short text) - Current value
  • Change (Short text) - Change percentage
  • Direction (Short text with predefined values: up, down, neutral)
  • Order (Number) - Display order in ticker

Expert Opinion Model

  • Quote (Long text)
  • Author Name (Short text)
  • Author Title (Short text)
  • Author Image (Media)
  • Publication Date (Date & time)

Featured Analysis Model

  • Title (Short text)
  • Category (Short text)
  • Summary (Long text)
  • Article Link (Reference to Article)
  • Premium (Boolean)
  • Publication Date (Date & time)

4. Generate API Keys

Go to Settings > API Keys and create a new API key. You'll need:

  • Space ID
  • Content Delivery API - access token

Important: Keep your API keys secure. Never commit them directly in your code.

Netlify Deployment

Follow these steps to deploy your website to Netlify:

1. Push Your Code to GitHub

Create a new repository on GitHub and push your code.

2. Connect to Netlify

  1. Sign up/log in to Netlify
  2. Click "New site from Git"
  3. Select GitHub and authorize Netlify
  4. Choose your repository

3. Configure Build Settings

  • Build command: npm run build
  • Publish directory: dist

4. Set Environment Variables

Add your Contentful API keys as environment variables (see next section).

5. Deploy Your Site

Click "Deploy site" and wait for the build to complete.

6. Connect Your Domain

  1. Go to Site settings > Domain management
  2. Click "Add custom domain"
  3. Enter your domain (e.g., fershman.com)
  4. Follow Netlify's instructions to update your DNS settings
  5. Enable HTTPS once DNS is verified

Environment Variables

The website uses environment variables to securely connect to Contentful. In Netlify, add these variables:

CONTENTFUL_SPACE_ID=your_space_id_here
CONTENTFUL_ACCESS_TOKEN=your_access_token_here

For local development, you can create a .env file with these variables.

Content Management Guide

Once your Contentful space is set up, you can manage your content without touching code:

Adding New Articles

  1. Log in to Contentful
  2. Go to Content > Add entry > Article
  3. Fill in all required fields
  4. Set the category to determine where it appears on the site
  5. Toggle "Featured" to show it on the homepage
  6. Click Publish

Updating Market Data

  1. Go to Content > Market Data
  2. Select the entry you want to update
  3. Change the values, direction, etc.
  4. Click Publish

Adding Expert Opinions

  1. Go to Content > Add entry > Expert Opinion
  2. Add the quote, author details, and image
  3. Click Publish

Triggering Builds

When you publish content in Contentful, your website should automatically rebuild if you've set up webhooks:

  1. In Netlify: Site settings > Build & deploy > Build hooks
  2. Create a new build hook named "Contentful Publish"
  3. Copy the URL
  4. In Contentful: Settings > Webhooks
  5. Create a webhook with the Netlify build hook URL
  6. Set it to trigger on Publish/Unpublish events

Development Guide

For developers who want to modify the website:

Local Development

# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build

Customizing the Design

  • Main styles are in assets/css/styles.css
  • Mobile responsive styles are in assets/css/mobile.css
  • Color scheme can be modified by changing the CSS variables at the top of styles.css

Adding New Pages

  1. Create a new HTML file in the appropriate directory
  2. Copy the structure from an existing page
  3. Update the content and links
  4. Add any necessary JavaScript in main.js or contentful.js

Note: If you modify the Contentful content models, you'll need to update the corresponding code in contentful.js to match your new structure.
